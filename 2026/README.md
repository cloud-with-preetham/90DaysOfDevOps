# 90 Days of DevOps - 2026 Edition

A comprehensive DevOps learning journey covering Linux, Shell Scripting, Git, Docker, and more.

## 📚 Overview

This repository contains my 90-day DevOps learning journey with hands-on tasks, projects, and documentation. Each day focuses on building practical skills required for modern DevOps engineering.

## 🗓️ Learning Path

### Phase 1: Linux Fundamentals (Days 1-15)

- **Day 01**: Introduction to DevOps and Cloud - Personal learning plan
- **Day 02**: Linux Architecture - Understanding the OS structure
- **Day 03**: Linux Commands - Essential command-line operations
- **Day 04**: Linux Practice - Hands-on command exercises
- **Day 05**: Linux Troubleshooting - Debugging and problem-solving
- **Day 06**: File I/O Operations - Working with files and streams
- **Day 07**: Linux File System - Directory structure and scenarios
- **Day 08**: Cloud Deployment - Deploying applications with Nginx
- **Day 09**: User Management - Creating and managing users/groups
- **Day 10**: File Permissions - Understanding and setting permissions
- **Day 11**: File Ownership - Managing file ownership with chown
- **Day 12**: Linux Revision - Consolidating core concepts
- **Day 13**: LVM (Logical Volume Manager) - Disk management
- **Day 14**: Networking Basics - Essential networking commands
- **Day 15**: Networking Concepts - Deep dive into networking

### Phase 2: Shell Scripting (Days 16-21)

- **Day 16**: Shell Scripting Basics - Variables, conditions, user input
- **Day 17**: Loops and Arguments - For loops, script arguments
- **Day 18**: Functions and Best Practices - Modular scripting
- **Day 19**: Automation Project - Backup, health checks, log rotation
- **Day 20**: Log Analysis - Building a log analyzer tool
- **Day 21**: Shell Scripting Cheatsheet - Quick reference guide

### Phase 3: Version Control with Git (Days 22-28)

- **Day 22**: Git Fundamentals - Basic Git operations
- **Day 23**: Git Branching - Working with branches
- **Day 24**: Git Collaboration - Merging and conflict resolution
- **Day 25**: Git Advanced - Rebasing and advanced workflows
- **Day 26**: Git Best Practices - Professional Git usage
- **Day 27**: Git Scenarios - Real-world problem solving
- **Day 28**: Git Revision - Consolidating Git knowledge

### Phase 4: Docker & Containerization (Days 29-37)

- **Day 29**: Docker Basics - Introduction to containers
- **Day 30**: Docker Images - Working with images
- **Day 31**: Dockerfile - Building custom images
- **Day 32**: Docker Volumes & Networking - Data persistence and networking
- **Day 33**: Docker Compose - Multi-container applications
- **Day 34**: Docker Compose Advanced - Complex deployments
- **Day 35**: Multi-stage Builds & Docker Hub - Optimization and registry
- **Day 36**: Docker Project - End-to-end containerized application
- **Day 37**: Docker Revision - Cheatsheet and review

### Phase 5: YAML & Configuration (Days 38-40)

- **Day 38**: YAML Fundamentals - Syntax and structure
- **Day 39**: Configuration Management
- **Day 40**: Advanced YAML

## 🛠️ Technologies Covered

- **Operating Systems**: Linux (Ubuntu/Debian)
- **Shell**: Bash scripting
- **Version Control**: Git & GitHub
- **Containerization**: Docker, Docker Compose
- **Networking**: TCP/IP, DNS, HTTP/HTTPS
- **Tools**: Nginx, systemd, cron, LVM
- **Languages**: YAML, Bash
