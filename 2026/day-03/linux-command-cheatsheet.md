# Day-3 Linux Commands Practice

**Note**: All command discription is referred to **man** pages in linux terminal.

## Cheatsheet of Linux Commands:
### 1. Process Management:
- `ps` - Report a snapshot of the current processes.
- `top` - Display Linux processes.
- `htop` - Interactive process viewer, same as **top** but interactive.
- `atop	` - Advanced System & Process Monitor.
- `uptime` - Tell how long the system has been running.
- `fuser` - identify processes using files or sockets
- `kill` - send a signal to a process.
- `nohup` - run a command immune to hangups, with output to a non-tty.
- `free` - Display amount of free and used memory in the system.
- `vmstat` - Report virtual memory statistics.

### 2. File System:
- `ls` - list directory contents
- `cd` - Change the shell working directory.
- `pwd` - print name of current/working directory.
- `mkdir` - make directories.
- `rmdir` - remove empty directories.
- `cp` - copy files and directories.
- `mv` - move (rename) files.
- `rm` - remove files or directories.
- `wc` - print newline, word, and byte counts for each file.
- `touch` - change file timestamps.

### 3. Networking Commands:
- `ping` - send ICMP ECHO_REQUEST to network hosts.
- `ip addr` - show / manipulate routing, network devices, interfaces and tunnels.
- `traceroute` - trace the route to a host.
- `mtr` - a network diagnostic tool.
- `nslookup` - query Internet name servers interactively.
- `hostname` - Hostname is used to display the system's DNS name.
- `dig` - DNS lookup utility.
- `curl` - transfer a URL.
- `whois` - client for the whois directory service.
